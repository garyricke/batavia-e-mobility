End Card
How-To

Every video closes with a "DID YOU KNOW?" end card — a polaroid of the kid plus the matching ordinance fact, with a side character in the lower-left. This page walks you through producing one. About 10 minutes per card once you've done a few.

9:16 · 2K nano-banana 10 min/card
01

Know the cast

Cross-check the kid in the video against this sheet every time. Several names look or sound similar — Sammy / Sammi, Bri / Brie. Getting the name right matters more than getting it fast.

02

Pick the side character

Every end card has one side character as a sticker cut-out in the lower-left. Choose based on the rule the episode teaches.

Officer Webb

Default · Enforcement

Use for any rule tied to enforcement, citations, traffic law, or a kid getting stopped.

Webb & River

Required · K9 episodes

Required whenever River appears in the video, even briefly. River is a female yellow Lab — never depict her as male.

Matt

Instructional · Teacher voice

Use for rules about safe behavior, decision-making, habits — "what would your teacher say" energy, not enforcement.

Lori

City programs · Parents

Use for rules about city programs, registration, parent-facing info, community resources, where to find the law.

Rotate the pose every time

Do not ship two end cards with the same side-character pose. Same face, same uniform, different action. Examples to draw from:

  • Webb: pulling sunglasses down · pointing index finger up · hand on hip with badge prominent · dangling jail keys (used on Johnny) · thumbs-up · arms crossed
  • Matt: pointing at the tip box (used on Tyler) · arms crossed · finger to temple "think about it" · holding a skateboard · open-palm "stop" · two thumbs up
  • Lori: speaking gesture with raised hand · holding a city brochure · clipboard in hand · open-palm presenting · finger raised "important" · warm wave
03

Make the Pixar character

Pause the video on a frame where the kid is facing camera with a clear expression. Crop tight (chest-up). Save as PNG. Heads up: macOS Screenshot filenames have a non-breaking space — copy/rename to something like /tmp/kid-screencap.png before tools touch it.

Run nano-banana edit_image on the screencap. Save the output to generated_imgs/pixar-<name>.png (lowercase).

image: screencap.png aspect: 3:4 resolution: 2K
Transform this kid into a Pixar / Disney 3D animated character while keeping the EXACT same composition and crop as the source photo: tight chest-up portrait, same camera angle and zoom level. Preserve their real likeness precisely — [HAIR COLOR + STYLE], [EYE COLOR], [DISTINCTIVE FEATURES like freckles / braces / glasses], wearing [WHAT THEY ACTUALLY WORE — t-shirt color, helmet only if they had one on, etc.]. Stylize with the classic Pixar look: large expressive eyes with bright catchlights, soft subsurface skin shading, slightly stylized friendly proportions, clean cinematic 3D rendering, polished feature-film aesthetic. Background: soft-blurred sunny park or schoolyard with green trees and warm natural daylight, shallow depth of field. Vertical 3:4 framing matching the original.

Watch out for

  • Don't add a helmet if the kid wasn't wearing one in the screen grab. The model auto-adds helmets when prompted carelessly.
  • Don't change the composition. Match the source crop. Background changes are fine, but if the source is chest-up the Pixar version stays chest-up.
  • Compare against the canonical photoreal-Pixar style (Sammy, Tyler, Hadley). It should look like a Pixar feature still — not cel-shaded, not flat illustration.
  • If features feel off, retry with sharper details. Don't ship a "close enough" portrait — it'll bug the kid forever.
04

Bake the end card

Run nano-banana edit_image on the canonical template generated_imgs/end-card-hadley.png with the kid's new Pixar character as a reference image. The template carries the whole visual system — typography, washi tape, stamp, tip box, CTA — so all you're swapping is photo, label, tip text, side character.

Writing the label & tip box

Polaroid label format: [Name] — [short wry tag]. Keep it under ~25 characters total. The voice is relief-reflection — the kid is glad they learned the rule. Not snark, not Judy-style "anyway, try my pancakes" cadence. Look at the shipped cards for the right tone.

Tip box: paragraph 1 cites the rule under Ord. 2026-010 in plain language. Paragraph 2 is the kid-friendly takeaway that echoes the relief line from the video. Keep both short — the box auto-wraps and overflow eats into the side character.

Review every output

  • Hair color drift — the model sometimes darkens blondes. Re-prompt with the hair color emphasized if it shifts.
  • Doubled accessories — two sets of headphones, two helmets. Specify "ONE set" explicitly if it happens.
  • Garbled text on badges or the CTA. The pill must read bataviail.gov →. If it shows anything else, retry.
  • Side-character drift — wrong character or wrong pose. Retry with a sharper prompt and name the character explicitly.
  • River anatomy — she is female. If the K9 version shows male anatomy, re-prompt: "clean smooth underbody, female dog, no visible anatomy."

Save the final to generated_imgs/end-card-<name>.png.

05

What "right" looks like

These cards are the established baseline. Compare your output against them for typography, layout, color, and overall polish. If your card doesn't feel like it belongs in this set, don't ship it — retry.

The short list

  • Check the cast sheet if you're unsure of a name.
  • Use Webb & River any time River is in the video.
  • River is female — never depict her as male.
  • Vary the side-character pose every single card.
  • Don't add a helmet the kid wasn't wearing.
  • Don't change the composition of the kid's source crop.
  • Confirm the CTA reads bataviail.gov →.
  • Ship the card, then ping Gary for review before the episode goes out.
